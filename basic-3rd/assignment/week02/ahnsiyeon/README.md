# 과제
- week02 파일 안에 본인 이름 파일을 만든 후 그 안에 제출.

## Wireshark 심화

### 제출 파일
- Wireshark 패킷 캡처 파일 (`.pcapng`)
- 결과 정리 파일 (`.md` 또는 `.txt`)

### 목표
- Wireshark를 이용하여 PDF 파일 다운로드 패킷 캡처
- HTTP 상태 코드 분석

### 실습 과정
1. Wireshark 실행
2. 패킷 캡처 시작
3. 디스플레이 필터에 `HTTP` 설정
4. 용인시 홈페이지 접속  
 (https → http로 변경하여 암호화 해제)
5. 홈페이지에서 PDF 파일 다운로드  
 (다운로드 전체 과정 패킷 캡처)
 - http 환경에서는 경고창이 뜨면 "계속" 클릭
6. 패킷 캡처 종료 후 저장

### 분석 내용
- `통계 → HTTP → Packet Counter → HTTP Response Packets`
- 주요 상태 코드(200, 301, 404 등) 확인 및 의미 간단히 정리
1xx: 정보, 요청을 수락했으며 처리 중.
2xx: 성공, 정상 응답. 요청이 정상적으로 처리 됨
3xx: 리다이렉션, 요청을 완료하려면 추가 동작이 필요함
4xx: 클라이언트의 잘못된 요청 (ex. 404 not found)
5xx: 서버 오류, 서버 문제로 요청 처리 실패

### 결과 정리 (필수)
- 사용한 필터 - HTTP
- 다운로드 요청 패킷 설명 
파일을 다운로드할 때 HTTP GET 요청 패킷이 전송됨
(Get 요청: 서버에게 특정 파일 전송 요청)
패킷 확인 결과, 서버로 전송된 GET 요청에 대한 응답으로 200 OK 응답을 반환
-> 파일이 정상적으로 다운로드되었음을 확인 가능

- 확인한 HTTP 상태 코드 + 의미
패킷 분석 결과, HTTP Response 패킷은 총 21개였으며, 모든 응답이 200 OK(클라이언트 요청이 정상 처리됨).
-> 서버와 클라이언트 간의 통신이 정상적으로 이루어졌음을 확인
